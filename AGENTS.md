# ppt 开发指引

## 项目

`ppt` 是 Python 3.11+ CLI：用 OSS 交易账本为中美混合永久投资组合生成三级优先的自动均衡方案。它处理真实持仓与交易记录；README 是行为规约，任何会改变命令、数据格式或金融计算语义的改动都必须同步更新测试和 README。

## 开发约定

- 保持分层：计算层纯函数（无 IO、无输出、不读取环境或文件）；IO 层处理 OSS、行情与缓存；展示层只组装原子组件；CLI 只做参数、编排和异常处理。
- 所有终端输出经 `ppt.display` 的组件；不要在 CLI 或业务逻辑中散写 `print(Text(...))`。
- 固定四桶 25%，严格保持“四桶 > 桶内标的 > 美元/人民币”的优先级；不为目标外旧命令或旧数据格式保留兼容层。
- 保持交易单位、人民币估值、历史汇率与浮点容差；涉及有符号批次和自动均衡时覆盖边界和失败路径。
- 不覆盖或回退已有用户改动；不要把密钥、OSS 凭据或真实持仓写入仓库。

## 验证

```bash
uv run pytest
uv run ruff check .
```

需要本地开发安装时：`uv sync --locked --extra dev`。CLI 入口为 `uv run ppt`；实现细节与命令契约见 `README.md`。

## Flywheel（Codex）

按任务意图选择入口：

- 新需求、方案或明确要求实现：`/fwp-design <需求>`
- 具体错误、回归或持续集成故障：`/fwp-debug <问题>`
- 当前质量巡检（会运行项目检查）：`/fwp-inspect`
- 项目演进自查（宏观审查，不运行项目）：`/fwp-check`
- 恢复已有议题、拉取请求或中断交付：`/fwp-resume`

只要求报告或方案时，在对应阶段停止；原请求明确要求修复、实现或交付时，才继续进入后续阶段。`fwp-deliver` 是内部交付引擎，不直接调用。远端写操作与合并始终受用户授权和本仓库规则约束。
