# 目标

## 核心结果

- 为单个投资者维护一份 OSS 持仓账本，并生成自动均衡与新增资金合并后的交易方案。
- 长期目标固定为四桶各 25%；分配严格遵循：**四大桶均衡 > 桶内标的均衡 > 美元/人民币均衡**，后一级不得让前一级变差。
- 只保留 `init`、`buy`、`status`、`plan`、`history` 五个命令。

## 必需功能

- `ppt init`：经确认后先在 OSS 备份当前账本，再把持仓和交易历史重置为空；没有旧账本时直接创建。
- `ppt buy 代码#股数@价格 [...]`：一次原子记录多个标的。股数为正表示买入，为负表示卖出；任一参数无效或卖出超过持仓时整批不写入。
- `ppt status`：读取账本、当前有效行情和 USD/CNY，展示组合总市值、七个固定标的的持仓与人民币估值、四桶当前占比及走廊状态、桶内占比、美元/人民币占比和三级偏差；不展示交易历史或交易建议，不写账本。
- `ppt plan <人民币金额>`：自动读取账本、有效行情和 USD/CNY，在新增资金、卖出所得与交易单位约束内生成净交易方案：
  1. 四桶目标各 25%；固定走廊为 15%–35%，只有越过走廊才产生存量跨桶卖出均衡，走廊内优先用新增资金修正偏差。
  2. 桶内标的目标为等市值；新增买入优先较低配标的，明显失衡时允许在桶内换仓。
  3. 前两级同样最优时，再使全部资产的美元与人民币权重接近 50%/50%。
- `plan` 将存量均衡、桶内均衡、币种均衡、`GLDM → 518880.SS` 和 `SGOV → 511360.SS` 换仓及新增买入按标的合并为净股数。正数买、负数卖、零净额删除；同一标的最多出现一次，不能同时买卖。
- `plan` 输出净交易数量、预计人民币收支、未使用金额、买入前后三级偏差，以及一条完全一致、可直接复制执行的 `ppt buy ...` 命令；买入成本不得超过新增金额与卖出所得之和，`plan` 本身不写账本。
- `plan` 展示近期趋势方向和相关性异常，仅作为提示，不改变 25% 目标、走廊或三级分配顺序。
- `ppt history`：按批次倒序展示时间、标的、带正负号的股数和成交价；顶部汇总累计投入、累计取出、当前市值、收益额和收益率。
- OSS 写入必须串行且原子；覆盖前必须完成备份，锁、备份或写入失败时保持原账本不变。

## 输入与输出

- 固定标的：股票（SPYM、AVUV）、债券（VGIT）、黄金（GLDM、518880.SS）、现金（SGOV、511360.SS）。
- 美股按 1 股交易，A 股按 100 份交易；价格使用标的原币，估值与资金约束统一按有效 USD/CNY 换算成人民币。
- 缺失、非正或非有限价格、汇率与预算必须报错；交易股数不能为零，卖出后持仓不能为负。
- 所有结果都在终端明确展示；只有 `init` 和 `buy` 修改账本。

## 明确不做

- 不使用风险平价，不动态改变四桶 25% 的长期目标。
- 不提供独立的 `sell`、`undo`、`rebalance`、`config` 或清理命令。
- 不连接券商，不自动下单，不支持自定义标的、目标比例、GUI、Web 服务或多用户。
- 不保留未写入本目标的旧命令、旧配置、旧数据格式或兼容层。

## 运行完成标准

- 从干净检出执行 `uv sync --locked --extra dev` 成功；`uv run ppt --help` 只展示 `init`、`buy`、`status`、`plan`、`history`。
- 在隔离 OSS 账本和固定行情下真实执行：`init` → 多标的正负数 `buy` → `status` → `plan 10000` → 执行其输出命令 → `history`；重启后结果不变。
- `status` 完整展示七个标的、四桶、桶内和币种的当前状态；空持仓不发生除零或伪造占比，调用前后账本内容、备份和锁状态不变。
- 使用明显偏配持仓运行 `plan`：走廊、三级优先级和换仓均生效；所有来源按标的净额合并，每个标的最多一次，买入成本不超过新增资金加卖出所得。
- `history` 的批次记录和收益汇总与账本、行情一致；趋势与相关性只产生提示，不改变交易方案。
- 再次 `init` 前生成可恢复备份；超卖、部分无效批次、无效行情、并发写入和备份失败均保持原账本不变。
- `uv run pytest`、`uv run ruff check .` 和上述真实命令全部通过，没有未处理异常、RuntimeWarning、虚假成功输出或遗留锁。
